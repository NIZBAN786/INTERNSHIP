1. Objective

Perform a TCP SYN port scan (Nmap) against the local network (192.168.1.0/24), identify hosts and open ports, and document findings according to the internship brief.

2. Tools and Environment

  • Tool: Nmap v7.95
  • Environment: Kali Linux (or equivalent), connected to the local network 192.168.1.0/24
  • Output Saved: nmap-test.txt

3. Scan Command

nmap -sS -Pn -vv -oN nmap-test.txt 192.168.1.0/24
  • -sS: TCP SYN (stealth) scan
  • -Pn: Treat all hosts as online (skip host discovery ping)
  • -vv: Very verbose output
  • -oN: Normal output to file

5. Analysis

  1. Host 192.168.1.1:

    • Open ports:
      • 23/tcp (telnet): Unencrypted remote login service—highly insecure; should be disabled or replaced by SSH (port 22).
      • 80/tcp (http): Standard web service—investigate web application for vulnerabilities.
      • 3517/tcp (802-11-iapp): Wi‑Fi Inter‑Access Point Protocol—likely related to wireless AP management; review for unintended exposure.
    • Recommendation: Disable Telnet, restrict HTTP access via firewall, and verify wireless management protocols are secured.

  2. Host nizban-s-a26 (192.168.1.4):

    • No open ports detected—appears hardened or host may be down for services.
    • Recommendation: Confirm intended service configuration; validate host purpose and firewall rules.

6. Conclusion

The SYN scan identified two active hosts. One host (192.168.1.1) exposes insecure services (Telnet) and a wireless management port that require immediate attention. The second host appears closed to external scans. Further steps may include vulnerability scanning (e.g., -sV for version detection), service enumeration, and deeper analysis with Wireshark if necessary.
